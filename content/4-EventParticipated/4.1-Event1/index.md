## Event Objectives

The purpose of this event was to help students and young developers improve their learning mindset in the AI era. The event focused on how to build better self-learning habits, use AI tools effectively, write better prompts, and prepare the mindset of an AI-ready fresher.

Another important objective of the event was to introduce a structured way of working with AI agents. Instead of using AI only for simple answers, participants learned how AI can support software development when it is guided by clear context, proper documentation, and well-defined tasks.

## Key Highlights

One of the main topics of the event was how to make learning more consistent and effective. The speakers explained that students often lose motivation because learning does not always provide immediate rewards. Therefore, it is important to build good habits, create a suitable learning environment, and set clear goals.

The event also focused on prompt engineering. I learned that an effective prompt should include role, instruction, context, input, expected output format, examples, and constraints. This helps AI tools provide more accurate and useful responses.

Another important part was about the mindset of an AI-ready fresher. The speakers emphasized that AI should be used as a tool to support productivity, not as a replacement for real understanding. A developer still needs to understand the problem, analyze the solution, and verify the output generated by AI.

The event also introduced the BMAD/BMX method, which is a document-driven approach to working with AI agents. This method focuses on writing clear documents, dividing the project into epics and user stories, and using different AI agents such as PM Agent, Architect Agent, Developer Agent, and Review Agent to support the development process.

## Key Takeaways

- Effective learning requires both discipline and a good system.
- Prompt engineering is an important skill when using AI for studying and software development.
- AI can increase productivity, but the core understanding must still come from the developer.
- A good developer should understand the problem, choose a suitable solution, and know how to maintain it in the long term.

## Application to My Project

The knowledge from this event can be applied directly to my **RoughLife** game project. When working with Unity, C#, online multiplayer, or AWS services, I can use AI more effectively by providing clear context, describing the problem carefully, and checking the result instead of copying it blindly.

The document-driven mindset is also useful for my team because RoughLife includes many components such as gameplay, boss mechanics, online co-op, AWS architecture, and deployment. Clear documentation helps reduce confusion and makes the project easier to maintain.

## Event Experience

This event was meaningful because it combined technical knowledge with career mindset. I not only learned how to use AI better, but also understood the importance of integrity, teamwork, long-term thinking, and continuous improvement as a future developer.
