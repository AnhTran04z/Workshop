## Event Objectives

The purpose of this event was to provide participants with practical knowledge, personal development insights, and real-world sharing related to AWS learning, hackathon experience, confidence building, and DevOps thinking.

This event was meaningful because it did not focus only on cloud technology. It also included communication skills, teamwork, self-confidence, project experience, and lessons learned from real development activities.

## Key Highlights

The first session was presented by **Huynh Thai Linh** with the topic *Level Up Your AWS Skills with Cloud Quest and Floci*. This session helped participants understand how AWS learning can become more interesting and effective through gamified learning and structured practice. It encouraged learners to improve their cloud knowledge step by step instead of feeling overwhelmed by many AWS services.

Another memorable part of the event was the hackathon sharing by **The Ballers Team**, including **Huynh An Khuong, Mai Quoc Anh, and Nguyen Tran Minh Quan**. Their topic showed the real experience of joining a hackathon, including the fun parts, bugs, pressure, back pain, deep questions, and teamwork challenges. Through this session, I understood that a successful project is not only about the final result, but also about the process of solving problems together under time pressure.

The session by **Nguyen Thi Quynh Nhu**, *Why We Always Need Confidence*, focused on the importance of confidence in career development, campus life, and personal growth. This topic reminded me that technical knowledge alone is not enough. Confidence is also important when communicating, presenting, working in a team, and taking initiative.

Another interesting presentation was *Tu Vi Dai Viet* by **Nghia Tran**. This topic brought a creative and different perspective to the event. It showed that a presentation can be engaging when the speaker connects ideas with culture, personal identity, and storytelling.

The final highlighted session was *The Hidden Iceberg of a Project: DevOps Before Disaster* by **Tran Minh Quan**. This session was very useful because it explained that many project problems are hidden beneath the surface. Before a system fails, there are often warning signs related to deployment, infrastructure, monitoring, workflow, or team coordination. This helped me understand that DevOps is not only about tools, but also about preparation, prevention, and long-term project stability.

## Key Takeaways

- Technical growth also requires confidence, teamwork, communication, and real project experience.
- AWS learning should be built progressively through hands-on practice.
- Hackathon work often includes pressure, bugs, limited time, and teamwork challenges.
- DevOps helps reduce project risks through preparation, monitoring, and prevention.

## Application to My Project

The knowledge from this event can be applied directly to my **RoughLife** project. The AWS-related sharing motivated me to continue improving my cloud foundation, which is useful for deploying and managing parts of the game system in the future.

The hackathon experience also gave me useful lessons for teamwork and project execution. Since RoughLife is a group project, communication, task coordination, and problem-solving under pressure are very important. The confidence session reminded me that presenting ideas clearly and speaking with confidence are necessary, especially when explaining project architecture, gameplay systems, or technical choices.

The DevOps session is also highly relevant to RoughLife. The project is not only a Unity game but also a system that can involve multiplayer networking, deployment, monitoring, and cloud services. Understanding hidden risks before failure happens can help my team design a more stable and maintainable project.

## Event Experience

This event was a valuable experience because it combined technical knowledge, practical project lessons, and personal development topics in one program. I enjoyed the diversity of the sessions because each speaker brought a different perspective, making the event both informative and inspiring.

Overall, this event helped me improve not only my technical understanding but also my mindset as a student preparing for a future career in technology. It gave me more motivation to keep learning AWS, improve teamwork skills, and develop my RoughLife project in a more practical and professional direction.
