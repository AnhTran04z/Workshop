During my participation in the **First Cloud AI Journey** program, I had the opportunity to learn practical knowledge about cloud computing, especially AWS services and how they can be applied to a real project. Through the process of preparing reports and setting up AWS services for the **RoughLife** online game system, I gained a better understanding of how to build a cloud-based backend architecture for a real product.

The work I completed included storing the game build with Amazon S3, distributing files through CloudFront, protecting web traffic with AWS WAF, preparing player authentication with Cognito, storing data with DynamoDB, deploying the Lambda Room API, exposing the API through API Gateway, and monitoring the backend with CloudWatch.

### Overall Evaluation

**1. Learning and Working Environment**  
The learning environment of the program is friendly, open, and suitable for students who want to learn about AWS and cloud computing. The materials, workshops, and hands-on approach helped me learn the theory and apply it directly to my own project.  
What I appreciated most is that the program did not only introduce AWS services but also encouraged learners to apply them to real project scenarios.

**2. Support from Mentor / Team Admin**  
During the implementation process, the mentor and admin team provided helpful guidance, explained unclear points, and gave feedback to improve my report. The feedback helped me better understand how to present evidence, organize workshop content, and connect AWS services into an overall architecture.  
In particular, the comments on architecture diagrams, service setup, and report writing helped me improve the project in a more practical direction.

**3. Relevance of the Program to My Academic Major**  
The program content is highly relevant to my Information Technology major and my software development direction. Topics such as cloud computing, serverless backend, API, database, and monitoring are directly related to real software development.  
For the RoughLife project, these topics helped me understand how an online game can connect to a cloud backend to handle rooms, player data, and match results.

**4. Learning and Skill Development Opportunities**  
Through the program, I learned many new technical skills such as using the AWS Console, creating S3 buckets, configuring CloudFront, setting up WAF, creating Cognito User Pools, working with DynamoDB, deploying Lambda, creating API Gateway, and monitoring logs with CloudWatch.  
Besides technical skills, I also improved my reporting skills, evidence presentation, system architecture analysis, and debugging ability during deployment.

**5. Learning Culture and Community Spirit**  
The community spirit of the program is a positive point. Participants can learn from workshops, sample reports, and feedback from mentors or admins. This makes the program not only a place to learn AWS but also a place to practice real product implementation thinking.  
Being able to refer to different implementation approaches also helped me broaden my perspective when designing the architecture for my own project.

**6. Program Organization and Learning Structure**  
The program structure is suitable for students because the content is divided into workshops, stages, and specific evidence-based tasks. This makes it easier to track progress and understand what needs to be completed.  
However, if the program provides more detailed checklists or rubrics for each evaluation level, learners can more easily identify what they need to prepare to achieve better results.

### Additional Questions

**What did I find most satisfying during the program?**  
The most satisfying part was being able to apply AWS knowledge directly to my RoughLife game project. Instead of only learning theory, I was able to set up real AWS services and collect evidence for each step. This helped me understand the role of each service in a practical online system.

**What do I think should be improved for future participants?**  
In my opinion, the program could improve by adding more summary guidance after each group of services. For example, after learning S3, CloudFront, Lambda, API Gateway, and DynamoDB, a summary architecture diagram or sample request flow would help learners better understand how the services connect together.  
In addition, the program could include more real-world examples for different project types such as web applications, mobile applications, or online games so that learners can relate the content to their own projects more easily.

**Would I recommend this program to my friends? Why or why not?**  
I would recommend this program to friends who want to learn AWS and cloud computing through hands-on practice. The program is suitable for IT students because it helps learners gain practical knowledge, learn how to set up cloud services, and create clear evidence that can be used in reports or portfolios.

---

### Suggestions and Expectations

**Suggestions to improve the program experience**  
I suggest that the program provide a checklist for each workshop, including what screenshots should be captured, what information should be recorded, and what the expected output should be. This would help learners avoid missing important evidence when writing reports.

**Expectation to continue the program in the future**  
I would like to continue joining more advanced programs or workshops, especially topics such as Amazon GameLift, CI/CD, system security, advanced monitoring, and cloud cost optimization. These topics are very useful for developing real-world products.

**Other comments**  
Overall, the First Cloud AI Journey program helped me gain practical knowledge about AWS and cloud implementation for a specific project. After participating in the program, I feel more confident in reading AWS documentation, using the AWS Console, analyzing system architecture, and presenting technical evidence in reports.
